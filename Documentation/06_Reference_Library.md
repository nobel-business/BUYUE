# 06 — Reference Library
### Buyue (بيوع) — Inspiration & Benchmark Index

> **Purpose:** A structured, growing catalogue of reference websites that define the quality bar for Buyue's site — especially for animation, premium feel, layout, and interaction. Each reference is documented against a fixed template so comparisons are consistent.
>
> **Rules for this document:**
> - **Do NOT invent website names.** Only real, client-approved or team-vetted references may be entered.
> - The two references below (**azure.sa**, **above-limits.com**) are supplied by the client in the requirement file as the animation/feel benchmark.
> - Additional references are added over time using the **blank template** at the bottom. Leave placeholders (`[ … ]`) until a real reference is confirmed — never fill them with fabricated names or fabricated ratings.
> - References define **feel and quality**, not content or layout to copy. Buyue's own visual language (Doc 03/04) always governs the final design.

| Field | Meaning |

## Reference 01 — Azure

- **Website:** https://azure.sa/
- **Reason for Selection:** Client-provided benchmark (requirement file). A Saudi-market reference chosen for its animation and premium feel — directly relevant to Buyue's goal of a cinematic, locally-rooted, high-end experience.
- **UI Inspiration:** `[to observe]` — capture the interface polish, spacing generosity, and component refinement.
- **Animation Inspiration:** Primary benchmark for **motion quality** — smooth scroll, cinematic reveals, refined hover/interaction feel. Match the *smoothness and pacing*, expressed in Buyue's motion tokens (Doc 05).
- **Typography Inspiration:** `[to observe]` — note heading scale, Arabic/Latin pairing approach, and rhythm.
- **Layout Inspiration:** `[to observe]` — section structure, use of full-bleed panels, whitespace.
- **Hero Inspiration:** `[to observe]` — how the hero establishes tone and pacing on load.
- **Navigation Inspiration:** `[to observe]` — nav behavior on scroll, mobile menu, language handling.
- **Cards Inspiration:** `[to observe]` — module/card treatment and hover behavior.
- **Footer Inspiration:** `[to observe]` — footer composition and content density.
- **Scrolling Inspiration:** Key reason for selection — study the **scroll smoothness, reveal triggers, and pacing**. Adapt (don't copy) into Buyue's scroll system.
- **Overall Rating:** `[team to score /10 after live review]`
- **Notes:** Client-provided. This is a *feel* benchmark, not a layout to replicate. If the site is RTL/Arabic, note its RTL patterns specifically. Verify the site is live and representative before the design phase.

## Reference 02 — Above Limits

- **Website:** https://above-limits.com/
- **Reason for Selection:** Client-provided benchmark (requirement file). Chosen for its animation and premium, cinematic web experience — a bar for the *quality* of motion and interaction Buyue wants.
- **UI Inspiration:** `[to observe]` — capture the premium interface detailing and interaction craft.
- **Animation Inspiration:** Primary benchmark for **cinematic motion** — mask/scroll reveals, parallax, refined transitions, custom cursor feel. Match the sophistication, not the exact effects; map to Buyue's motion vocabulary (Doc 05).
- **Typography Inspiration:** `[to observe]` — large editorial type, kinetic text reveals, hierarchy.
- **Layout Inspiration:** `[to observe]` — editorial/agency composition, full-bleed media, whitespace.
- **Hero Inspiration:** `[to observe]` — how the hero creates immediate cinematic impact.
- **Navigation Inspiration:** `[to observe]` — minimal nav, transitions, menu overlay behavior.
- **Cards Inspiration:** `[to observe]` — project/case modules and their hover/reveal treatment.
- **Footer Inspiration:** `[to observe]` — footer as a closing statement / CTA moment.
- **Scrolling Inspiration:** Key reason for selection — study **scroll-linked storytelling, pinned moments, and reveal pacing**. Adapt into Buyue's scroll system with performance guardrails.
- **Overall Rating:** `[team to score /10 after live review]`
- **Notes:** Client-provided. Likely LTR/agency-style — when adapting, ensure everything is re-checked for **RTL Arabic** and for **performance on mid-range mobile** (heavy motion sites can be slow; Buyue must not inherit that). Feel benchmark only.

## Guidance for using references

1. **Extract principles, not pixels.** Identify *why* a reference feels premium (pacing, space, easing) and re-express it in Buyue's system.
2. **Always RTL-check.** Most premium references are LTR; verify every borrowed pattern works mirrored for Arabic.
3. **Always performance-check.** Heavy animation references can be slow; Buyue must keep the premium feel *and* the performance budget (Doc 05 §34, Doc 07).
4. **Stay on-brand.** A reference technique that fights Buyue's colors, type, or personality is rejected, however impressive.
5. **Keep it verified.** Only real, live sites enter this library. Placeholders stay empty until a real reference is confirmed — never fabricated.
